API Documentation

This documentation describes a simple message API that allows users to store a message and retrieve it later using a secure one-time token.

When a message is created, the system generates a unique token that can be used to retrieve that message exactly once. Tokens expire after 24 hours and are marked as used once the message has been accessed.

System Overview

The system consists of two primary components:

  • Message API – Handles creation and retrieval of stored messages.
  • Token Helpers – Utility functions responsible for generating and validating secure one-time tokens.

Quick Start

Once the server is running, the API can be accessed at: 

http://localhost:3000

Messages can be created using a POST request and retrieved later using the generated token.
