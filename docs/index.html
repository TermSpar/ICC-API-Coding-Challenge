Documentation Overview

This documentation describes an API with two endpoints. The first endpoint (POST) asks for user-provided information that then gets attached to a randomly generated one-time use token. This token can then be used by the second endpoint (GET) to access the information that was provided to the first endpoint. Tokens can only be used once, and they will automatically expire after 24-hours.
