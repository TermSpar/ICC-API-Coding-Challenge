Token Helper Functions

The token helper module provides utility functions used by the API to generate, store, and validate one-time tokens for retrieving messages.

Tokens are stored in MongoDB and follow these rules:

  • Each token may only be used once.
  • Tokens expire after 24 hours.
  • Invalid tokens return standardized error responses.

These helpers are imported by the API routes and ensure that token creation and validation logic is organized in one location.

Exported Functions

Function Type Description
hashToken() non-async function Hashes a token for storage in/comparison with the database.
createToken() async function Creates a secure token, stores it in the database, and returns the token string.
validateToken() async function Validates a token by ensuring that it exists, has not been used, and has not expired.

hashToken()

Hashes a token string using SHA-256.

Parameters

Field Type Description
token string The generated token string to be hashed.

Return Value

Field Type Description
hashedToken string The hashed token usable for safe storage in the database.

Implementation Details


function hashToken(token) {
    return crypto
        .createHash('sha256')
        .update(token)
        .digest('hex')
}

The hashToken() makes use of the crypto module's SHA-256 hash function to securely store the generated tokens in the database.

Note: this function is being exported because message.js will need to make use of it when storing its own token information.

createToken()

Creates a Token Mongoose document and stores it in the database.

Return Value

Field Type Description
token string The generated token string returned to the API.

Implementation Details


async function createToken() {
    const tokenString = generateToken()
    const hashedToken = hashToken(tokenString)

    const token = new Token({
        token: hashedToken
    })

    await token.save()
    return tokenString
}

The createToken() function must be async because it calls the (async) save() function on the new token.

Note: the hashedToken is stored in the database while the tokenString itself is returned to the user. This ensures that only the user has access to the actual token.

validateToken()

Determines whether a token provided by a user is valid.

Parameters

Field Type Description
providedToken string The token provided by the user in the link.

Return Value

Field Type Description
success boolean Indicates whether the token validation succeeds.
token object | null The validated token object from the database if validation succeeds. Null if validation fails.
error string | null Error message describing why validation failed. Null if validation succeeds.
name null This will only return if validation fails. Always null in the standardized error response to maintain consistent API shape.
email null This will only return if validation fails. Always null in the standardized error response to maintain consistent API shape.
message null This will only return if validation fails. Always null in the standardized error response to maintain consistent API shape.

Validation Steps

  1. Confirm the token exists in the database.
  2. Ensure the token has not already been used.
  3. Check whether the token has expired (24 hours).
  4. If valid, mark the token as used.

Implementation Details


const hashedToken = hashToken(providedToken)
let actualToken

try {
    actualToken = await Token.findOne({ token: providedToken })
} catch (err) {
    return tokenError(err.message)
}

The function first attempts to find the token in the database (which must be checked using a hashed version of the token since the hash is what is actually in the database). If the database query fails, a standardized error response is returned. Otherwise, the token is stored in the actualToken variable. 


if (!actualToken)
    return tokenError("This link could not be found.")

const ageInMs = Date.now() - actualToken.createdAt
if (ageInMs > TOKEN_EXPIRATION_MS)
    return tokenError("This link has expired.")

if (actualToken.used)
    return tokenError("This link has already been used.")

If the token cannot be found, has expired (const TOKEN_EXPIRATION_MS = 24 * 60 * 60 * 1000, i.e. 24 hours) or has already been used once, validation fails and an appropriate error message is returned. 


actualToken.used = true

try {
    await actualToken.save()
} catch (err) {
    return tokenError(err.message)
}

If the token is valid, it is marked as used and saved back to the database to enforce the one-time use rule.

Successful Response


{
    "success": true,
    "token": { ...token object... }
}

When validation succeeds, the response contains a true success flag and the validated token document.

Error Response


{
    "success": false,
    "error": "This link has expired.",
    "name": null,
    "email": null,
    "message": null
}

If validation fails, a standardized error response generated by tokenError() is returned.

Internal Helper Functions

Function Description
generateToken() Generates token string.
tokenError() Generates standarized error output.

generateToken()

Creates a cryptographically secure random token string using Node's crypto module.

The token is generated as 256 bits (32 bytes) of random data and encoded as a 64-character hexadecimal string.

Implementation


function generateToken() {
    return crypto.randomBytes(32).toString('hex')
}

The use of crypto.randomBytes() makes sure that the token cannot be easily predicted or replicated.

tokenError()

Generates a standardized error response when an invalid token is provided.

Parameters

Parameter Type Description
errorMessage string The message describing why token validation failed.

Implementation


function tokenError(errorMessage) {
    return {
        success: false,
        error: errorMessage,
        name: null,
        email: null,
        message: null
    }
}

In addition to the provided errorMessage, the json response always includes the name, email, and message fields so that the API response format remains consistent even when an error occurs.
