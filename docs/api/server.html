Server Setup

This document explains the setup of the Node.js server for the ICC API, including environment configuration, MongoDB connection, middleware, routes, and server startup.

Environment Variables

Variable Default Description
DB_HOST localhost Host address for MongoDB connection.
DB_NAME bollinger-test Name of the MongoDB database.
SERVER_HOST localhost Host address to start the server.
SERVER_PORT 3000 Port number for the server to listen on.

MongoDB Connection

The server connects to MongoDB using Mongoose. The connection URI is built from environment variables with defaults.

Implementation Details

Click to expand 

const mongoURI = `mongodb://${DB_HOST}/${DB_NAME}`
mongoose.connect(mongoURI)
const db = mongoose.connection

db.on('error', (error) => console.error('Database connection error:', error))
db.once('open', () => console.log(`Connected to Database: ${DB_NAME} at ${DB_HOST}`))

The mongoose.connect() method is called with the constructed mongoURI string.

db.on('error') logs any connection errors. db.once('open') confirms successful connection to the database.

Middleware

Express middleware is used to parse incoming JSON payloads.

Implementation Details

Click to expand 

app.use(express.json())

This allows the server to automatically parse JSON in request bodies, making req.body available in routes.

Routes

The server uses the messageRouter for all routes prefixed with /message.

Implementation Details

Click to expand 

const messageRouter = require('./src/routes/message')
app.use('/message', messageRouter)

All requests to /message are handled by the imported messageRouter.

Server Startup

The Express server is started using the specified host and port.

Implementation Details

Click to expand 

app.listen(SERVER_PORT, SERVER_HOST, () =>
  console.log(`Server started on http://${SERVER_HOST}:${SERVER_PORT}`)
)

The server listens on SERVER_HOST:SERVER_PORT and logs a message when startup is successful.
