Database Models

The application stores its data using MongoDB through the mongoose ODM (Object Document Mapper). Two models define the database structure:

Model Description
Message A specified message (250 character limit) with a name, email, and token attached.
Token A unique token that can be used to retrieve messages.

These schemas enforce the structure of documents stored in the MongoDB collections.

Message Model

Stores user-submitted messages along with the name, email, and the one-time token that allows retrieving the message later.

Fields

Field Type Required Description
success Boolean No Indicates whether the message was successfully created.
error String No Provides any error message.
name String Yes Name attached to the message.
email String Yes Email address attached to the message.
message String Yes The message content itself.
token String No One-time token string associated with the message.

Implementation


const mongoose = require('mongoose')

const messageSchema = new mongoose.Schema({
    success: {
        type: Boolean
    },
    error: {
        type: String
    },
    name: {
        type: String,
        required: true
    },
    email: {
        type: String,
        required: true
    },
    message: {
        type: String,
        required: true,
        maxlength: 250
    },
    token: {
        type: String,
    }
})

module.exports = mongoose.model('Message', messageSchema)

Note: The name, email, and message fields are the only ones required because their content is provided by the user. The success, error, and token fields will be populated by the API itself and so their requirement need not be enforced.

Token Model

Stores one-time tokens that allow users to retrieve messages. Each token is unique, can only be used once, and expires after 24-hours.

Fields

Field Type Default Description
token String None The unique token string itself.
createdAt Date Date.now Token creation date.
used Boolean false Indicates whether the token has already been used.

Implementation


const mongoose = require('mongoose')

const tokenSchema = new mongoose.Schema({
    token: {
        type: String,
        unique: true
    },
    createdAt: {
        type: Date,
        default: Date.now,
    },
    used: {
        type: Boolean,
        default: false 
    }
})

module.exports = mongoose.model('Token', tokenSchema)

Note: The expiration logic for tokens is handled in the tokenHelpers.js module rather than in the schema. This is because using expires would delete the object from the database, which would prevent informing the user that the token, in fact, expired.
