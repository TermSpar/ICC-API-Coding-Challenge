Message API

This API allows the user to generate a random token that has a name, email, and message attached to it. This token can then be used to retrieve the original name, email, and message.

Endpoints

GET /message/:token

Retrieves a message using a one-time token.

const providedToken = req.params.token

Extracts the token the user provided in the URL. 


const validToken = await validateToken(providedToken)
if (!validToken.success){
    res.status(400).json(validToken)
}

Calls the (async) validateToken() function from tokenHelpers.js, which will make the token accessible as validToken.token if it exists. If the token does not exist, validToken.success will return false along with an appropriate error message.

The status will be a 400 error to inform the user that the issue is on their end. All of this information will come back in a json format. 


try {
    message = await Message.findOne({ token: validToken.token.token })
} catch (err) {
    return res.status(500).json({ 
        success: false,
        error: err.message,
        name: null,
        email: null,
        message: null 
    })
}

Since findOne() is an async function that could hypothetically fail, it is wrapped in a try-catch statement. Any error will be returned to the user, along with a false success flag, and null values for name, email, and message.

Note: we must return this error otherwise the program would continue with a null message. 


res.status(201).json({
    success: true,
    error: null,
    name: message.name,
    email: message.email,
    message: message.message
})

If the token and its message have both been found, then the Message model object can be retrieved. This message will have its name, email, and message returned along with a true success flag and a false error flag.

Example Input

GET http://localhost:3000/message/eae18dc7e99990845817023567d7cdf03e50fbabb28c5c1432f50cbaf84feedf

Example Output

{
    "success": true,
    "error": null,
    "name": "Ben",
    "email": "Ben@test.com",
    "message": "This is Ben's link... hi!"
}

POST /message/

Creates a Message with a name, email, and message content. A one-time Token is also created and attached for secure retrieval. 

let token 
try {
    token = await createToken()
} catch (err) {
    return res.status(400).json({ 
        success: false,
        error: err.message,
        token: null
    })
}

Calls the (async) createToken() function from tokenHelpers.js, which generates a one-time token with a 24-hour expiration period. This token is put in the MongoDB.

Wrapped in a try-catch statement. If createToken() fails, its error will be caught and returned with a status 500 error. 

const message = new Message({
    name: req.body.name,
    email: req.body.email,
    message: req.body.message,
    token
})

A new Message Mongoose document is created with the name, email, and message provided by the user. The newly generated token is also attached. 

try {
    await message.save()
    res.status(201).json({
        success: true,
        error: null,
        token
    })
} catch (err) {
    res.status(400).json({ 
        success: false,
        error: err.message,
        token: null
    })
}

Attempts to save() the new message. If the user has entered correct input, the message will be saved, and a status 201 will return. The json data will contain a true success flag, a null error flag, and the generated token string.
