Message API

This API allows the user to generate a random token that has a name, email, and message attached to it. This token can then be used to retrieve the original name, email, and message.

Messages are stored in MongoDB and can only be retrieved once using their associated token. Tokens expire after 24 hours and are marked as used after successful retrieval.

Endpoints

Method Route Description
GET /message/:token Retrieves a stored message using a one-time token.
POST /message/ Creates a new message and generates a retrieval token.

GET /message/:token

Retrieves a message using a one-time token.

Path Parameters

Parameter Type Description
token string The one-time token used to retrieve a stored message.

Implementation Details

const providedToken = req.params.token

Extracts the token the user provided in the URL. 


const validToken = await validateToken(providedToken)
if (!validToken.success){
    res.status(400).json(validToken)
}

Calls the (async) validateToken() function from tokenHelpers.js, which will make the token accessible as validToken.token if it exists.

If the token does not exist, validToken.success will return false along with an appropriate error message. The status will be a 400 error to inform the user that the issue is on their end. All of this information will come back in a JSON format. 


try {
    message = await Message.findOne({ token: validToken.token.token })
} catch (err) {
    return res.status(500).json({ 
        success: false,
        error: err.message,
        name: null,
        email: null,
        message: null 
    })
}

Since findOne() is an async function that could hypothetically fail, it is wrapped in a try-catch statement. Any error will be returned to the user along with a false success flag and null values for name, email, and message.

Note: we must return this error otherwise the program would continue with a null message. 


res.status(201).json({
    success: true,
    error: null,
    name: message.name,
    email: message.email,
    message: message.message
})

If the token and its message have both been found, then the Message model object can be retrieved. This message will have its name, email, and message returned along with a true success flag and a false error flag.

Example Request


GET http://localhost:3000/message/eae18dc7e99990845817023567d7cdf03e50fbabb28c5c1432f50cbaf84feedf

Example Response

{
    "success": true,
    "error": null,
    "name": "Ben",
    "email": "Ben@test.com",
    "message": "This is Ben's link... hi!"
}

Status Codes

  • 200 – Message retrieved successfully
  • 400 – Invalid or expired token
  • 500 – Server error while retrieving the message

POST /message/

Creates a Message with a name, email, and message content.

A one-time Token is also created and attached for secure retrieval.

Request Body

Field Type Required Description
name string Yes Name of the message sender
email string Yes Email of the sender
message string Yes The message content

Implementation Details

let token 
try {
    token = await createToken()
} catch (err) {
    return res.status(400).json({ 
        success: false,
        error: err.message,
        token: null
    })
}

Calls the (async) createToken() function from tokenHelpers.js, which generates a one-time token with a 24-hour expiration period. This token is put in the MongoDB.

Wrapped in a try-catch statement. If createToken() fails, its error will be caught and returned with a status 500 error. 


const message = new Message({
    name: req.body.name,
    email: req.body.email,
    message: req.body.message,
    token
})

A new Message Mongoose document is created with the name, email, and message provided by the user. The newly generated token is also attached. 


try {
    await message.save()
    res.status(201).json({
        success: true,
        error: null,
        token
    })
} catch (err) {
    res.status(400).json({ 
        success: false,
        error: err.message,
        token: null
    })
}

Attempts to save() the new message. If the user has entered correct input, the message will be saved and a status 201 will return.

The JSON data will contain a true success flag, a null error flag, and the generated token string.

Example Request


POST http://localhost:3000/message
Content-Type: application/json

{
    "name": "Ben",
    "email": "Ben@test.com",
    "message": "This is Ben's link... hi!"
}

Example Response

{
    "success": true,
    "error": null,
    "token": "3845b90652c3e50552404b647319d4fac2e8997cef8453a87e2a2b2e36e719af"
}

Status Codes

  • 201 – Message successfully created
  • 400 – Invalid request data
  • 500 – Server error while generating token
