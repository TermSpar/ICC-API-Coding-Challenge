Server Setup

Setup for the Node.js server, its routes, and connection to the MongoDB.

Environment Variables

Variable Default Description
DB_HOST localhost Host address for MongoDB connection.
DB_NAME bollinger-test Name of the MongoDB database.
SERVER_HOST localhost Host address to start the server.
SERVER_PORT 3000 Port number for the server to listen on.

MongoDB Connection

The server connects to MongoDB using Mongoose. The connection URI is constructed from environment variables (with defaults).

Implementation


const mongoURI = `mongodb://${DB_HOST}/${DB_NAME}`
mongoose.connect(mongoURI)
const db = mongoose.connection

db.on('error', (error) => console.error('Database connection error:', error))
db.once('open', () => console.log(`Connected to Database: ${DB_NAME} at ${DB_HOST}`))

Middleware

Express.js middleware is used to parse incoming JSON payloads.

Implementation


app.use(express.json())

Routes

The server uses the messageRouter for all routes prefixed with /message. Any route other than /message will returned as a 404 error.

Implementation


const messageRouter = require('./src/routes/message')
app.use('/message', messageRouter)

app.use((req, res) => {
  res.status(404).json({
    success: false,
    error: `Route '${req.originalUrl}' not found`,
    suggestion: "Try '/message'"
  })
})

Server Startup

The Express server is started using the specified host and port.

Implementation


app.listen(SERVER_PORT, SERVER_HOST, () =>
  console.log(`Server started on http://${SERVER_HOST}:${SERVER_PORT}`)
)
