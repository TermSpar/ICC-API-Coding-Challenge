Application Initialization & Database Connection

The Express app is created and configured with middleware and routes. The MongoDB connection is established using Mongoose only when the app is executed directly (not when imported). This prevents unintended database connections during testing and allows the app to be reused in test environments.

Implementation


require('dotenv').config()
const express = require('express')
const app = express()
const mongoose = require('mongoose')

const DB_HOST = process.env.DB_HOST || 'localhost'
const DB_NAME = process.env.DB_NAME || 'bollinger-test'
const SERVER_HOST = process.env.SERVER_HOST || 'localhost'
const SERVER_PORT = process.env.SERVER_PORT || 3000

if (require.main === module) {
  const mongoURI = `mongodb://${DB_HOST}/${DB_NAME}`
  mongoose.connect(mongoURI)
  const db = mongoose.connection

  db.on('error', (error) => console.error('Database connection error:', error))
  db.once('open', () => console.log(`Connected to Database: ${DB_NAME} at ${DB_HOST}`))
}

Middleware & Error Handling

Express middleware is used to parse incoming JSON payloads and handle both request-level and application-level errors. This includes handling invalid JSON, routing errors, as well centralized global error handling.

Since the API's routes are wrapped by the asyncHandler() (read about that on the Additional Helper Functions page), all thrown errors will be caught here.

JSON Parsing Middleware


app.use(express.json())

JSON Syntax Error Handling


app.use((err, req, res, next) => {
  if (err instanceof SyntaxError && err.status === 400 && 'body' in err) {
    return res.status(400).json(formatErrorMessage("Invalid JSON format"))
  }
  next()
})

Global Error Handler


app.use((err, req, res, next) => {
  console.error(err)

  if (req.method == "GET") {
    if (err instanceof HttpError) {
      return res.status(err.statusCode).json(formatGETError(err.message))
    }

    if (err.name === 'ValidationError') {
      return res.status(400).json(
        formatGETError(err.message || "Internal Server Error", {
          token: null
        })
      )
    }

    return res.status(500).json(
      formatGETError(err.message || "Internal Server Error")
    )
  }

  if (req.method == "POST") {
    return res.status(500).json(
      formatPOSTError(err.message || "Internal Server Error")
    )
  }
})

Note: A distinction is made between POST and GET requests since their error formats must be different. Additionally, validation errors will only ever be raised on GET requests.

Routes

The server uses the messageRouter for all routes prefixed with /message. Any unsupported route results in a 404 response.

Implementation


const messageRouter = require('./src/routes/message')
app.use('/message', messageRouter)

app.use((req, res) => {
  res.status(404).json(formatErrorMessage(`Route '${req.originalUrl}' not found`, {
    suggestion: "Try '/message'"
  }))
})

Server Startup

The Express server is started using the specified host and port when the file is executed directly.

Implementation


app.listen(SERVER_PORT, SERVER_HOST, () =>
  console.log(`Server started on http://${SERVER_HOST}:${SERVER_PORT}`)
)

Testing

The Express app is exported so it can be used in the unit tests.

Additionally, the database connection is only initialized when the application is run directly (require.main === module). This ensures that tests do not unintentionally connect to MongoDB, allowing for easier mocking of data. 


module.exports = app
