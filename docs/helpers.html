Async Handler

Wraps an async Express route handler and automatically forwards any errors to the global error handling middleware (read about that on the Server Startup page).

This eliminates the need for repetitive try-catch blocks inside each route handler by catching rejected promises and passing them to next().

Implementation


function asyncHandler(fn) {
  return function (req, res, next) {
    Promise.resolve(fn(req, res, next)).catch(next)
  }
}

Example usage: 


router.get('/', asyncHandler(async (req, res) => {
  const data = await someAsyncOperation()
  res.json(data)
}))

Response Formatting

These helper functions ensure that all API responses follow a consistent structure. GET and POST requests use different formats to match the requirements of each endpoint.

GET Error Response

Generates a standardized error response for a GET request. The response always includes name, email, and message fields set to null when an error occurs. The extra field is for future development (e.g. if a suggested field were to be added for 404 errors) 


function formatGETError(message, extra = {}) {
  return {
    success: false,
    error: message,
    name: null,
    email: null,
    message: null,
    ...extra
  }
}

POST Error Response

Generates a standardized error response for a POST request. The response always includes success, error, and token fields. The error field is null if there are no errors, and token is null if there are errors. The extra field is for future development (e.g. if a suggested field were to be added for 404 errors) 


function formatPOSTError(message, extra = {}) {
  return {
    success: false,
    error: message,
    token: null,
    ...extra
  }
}

Success Response

Generates a standardized success response for retrieving a message. Since this will only be used by GET, it includes the Message's name, email, and message fields. 


function formatSuccessMessage(messageDoc) {
  return {
    success: true,
    error: null,
    name: messageDoc.name,
    email: messageDoc.email,
    message: messageDoc.message
  }
}

HttpError

Custom error class for representing HTTP-specific errors.

Extends the native JavaScript Error class by including an HTTP status code, allowing centralized error handling middleware to determine the appropriate response status.

Implementation


class HttpError extends Error {
  constructor(statusCode, message) {
    super(message)
    this.statusCode = statusCode
  }
}

Example usage: 


throw new HttpError(404, "Resource not found")
